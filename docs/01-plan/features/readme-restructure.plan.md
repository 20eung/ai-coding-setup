## 1. 사용자 의도 발견 (Phase 1)

### 1.1 핵심 문제

현재 문서는 "Professional Standard"를 목표로 작성되어 기업 보안(AWS Bedrock IAM), 자동화 마이그레이션 전략까지 포함했으나, 원래 의도는 **AI 코딩을 처음 해보는 사람을 위한 친절한 설정 매뉴얼**이었음.

- Claude Code, Gemini CLI 먼저 소개할 필요 없음
- **Antigravity로 쉬운 입문 → Claude Code/Gemini 심화** 순서가 맞음

### 1.2 대상 독자

| 그룹 | 특성 | 목표 |
|------|------|------|
| **팀원 (PART 1 대상)** | 네트워크 엔지니어. 터미널 익숙, 코딩은 오래전에 경험. AI 구성 처음 | Antigravity로 첫 AI 코딩 체험 |
| **협업자 (PART 2-3 대상)** | 소수. Claude Code + n8n 자동화를 함께 구현할 팀원 | n8n 워크플로우 자동화 구현 |

### 1.3 성공 기준

- **기본 단계**: PART 1을 따라 하면 20분 안에 Antigravity 설치 완료 + AI와 첫 대화
- **심화 단계**: PART 2-3을 따라 하면 Claude Code + n8n-MCP로 워크플로우 자동화 가능

## 2. 탐색한 대안 (Phase 2)

### Approach A: 2단계 구조 — 채택

한 문서 안에 "바로 시작하기 (PART 1)"와 "심화 확장 (PART 2-3)"으로 명확히 구분.

- **장점**: 파일 하나로 유지보수 단순, 독자가 원하는 파트만 선택 가능
- **단점**: 여전히 긴 문서, 스크롤이 많음
- **적합**: 팀 공유 단일 링크, 관리 부담 최소

### Approach B: 선형 진행 구조

입문 → 기초 → 심화로 흐르는 책 구조.

- **장점**: 자연스러운 학습 곡선
- **단점**: PART 구분이 없어 초보자가 어디서 멈춰야 할지 모름
- **미채택 이유**: 독자가 "여기서 멈춰도 OK" 신호를 받기 어려움

### Approach C: 두 문서 분리

BEGINNER.md / ADVANCED.md로 분리.

- **장점**: 대상별 최적화
- **단점**: 유지보수 부담 2배, 공유 링크 복잡
- **미채택 이유**: 소규모 팀에서 관리 오버헤드 과도

## 3. YAGNI 검토 (Phase 3)

### PART 1 (초보자 키트) — 포함 항목

| 항목 | 판단 | 비고 |
|------|------|------|
| Node.js/Git 설치 확인 | ✅ 포함 | Antigravity 전제 조건 |
| Git 기초 설정 (config + SSH) | ✅ 포함 | GitHub 연동 필요 |
| Antigravity 설치 + 첫 실행 | ✅ 포함 | **핵심 콘텐츠** |
| VS Code 확장 프로그램 설치 | ✅ 포함 (Antigravity 절 내부) | 별도 절 불필요 |
| Python 설치 | ⚠️ 간단 언급 | PART 2부터 필요, PART 1은 Node.js/Git만으로 충분 |

### PART 1에서 제외 항목 (PART 2-3으로 이동 또는 축소)

| 항목 | 판단 | 처리 방법 |
|------|------|----------|
| 리모트 서버 설정 (2.6) | ⚠️ 네트워크 엔지니어는 이미 앎 | "이미 아시면 건너뛰어도 됩니다" 주석 추가 |
| AWS Bedrock 상세 테이블 | ✂️ 과도한 분량 | 요약 Note로 축소 |
| Gemini 다중 계정 관리 | ✂️ 입문 불필요 | PART 2에서 간략히 |
| Python → n8n 마이그레이션 전략 | ✅ 유지 | PART 3 (협업자 대상) |

## 4. 새 문서 구조 설계

### 4.1 전체 목차 구조

```
# AI Coding Setup

## Quick Start — 지금 바로 시작! (→ PART 1 링크, ~20분)

## Key Features (간략화)

## 목차
  PART 1: 첫 AI 코딩 (모두 대상)
    1. 사전 준비
    2. Git 기초 설정
    3. Antigravity 설치 + 첫 AI 코딩 체험  ← 여기서 멈춰도 OK!

  PART 2: AI 코딩 환경 완성 (심화)
    4. Gemini CLI
    5. Claude Code (+ AWS Bedrock)
    6. GitHub 자동화 (gh)
    7. bkit

  PART 3: n8n 자동화 (고급)
    8. Docker 환경 구성
    9. n8n 설치 및 실행
   10. n8n-MCP + Skills
   11. Python → n8n 마이그레이션
   12. Claude Code로 n8n 개발

  라이선스 / 문의
```

### 4.2 주요 변경 사항

| 변경 전 | 변경 후 |
|---------|---------|
| Quick Start가 3줄 추상 안내 | "PART 1으로 바로 이동" 링크 + 예상 시간 안내 |
| Antigravity = 섹션 5 (후반부) | Antigravity = PART 1 섹션 3 (앞단으로 이동) |
| AWS Bedrock 4개 상세 테이블 | 1개 요약 Note + 외부 링크로 축소 |
| 섹션 번호 1-12 단순 나열 | PART 1 / PART 2 / PART 3 명확한 구분선 |
| "여기서 멈춰도 됩니다" 신호 없음 | PART 1 끝에 "✅ 여기서 멈춰도 됩니다!" 체크포인트 추가 |
| 리모트 서버 설정 (2.6) 필수처럼 서술 | "네트워크 엔지니어라면 이미 아실 내용, 건너뛰어도 됩니다" 주석 |

### 4.3 PART별 목표 독자와 시간

| PART | 독자 | 예상 시간 | 결과물 |
|------|------|----------|--------|
| PART 1 | 모든 팀원 | ~20분 | Antigravity로 첫 AI 대화 |
| PART 2 | 심화 희망자 | ~60분 | Claude Code + Gemini 독립 활용 |
| PART 3 | 자동화 구현자 | ~2시간 | n8n 워크플로우 자동화 구현 |
